# Z_DIAG — Standalone diagnostic tool for FLINT hardware

## Зачем создана

В FLINT 2.77 обнаружен баг: при повторной калибровке (homing) после первого прогона рисования, ось Z не доходит до датчика на ~4354 шага. Первый прогон работает идеально.

Из лога `FLINT32-12.03.2026_1.LOG`:
```
[START] POST-PROC k=1 lastn=0 xcur=22450 ycur=690 zcur=9096 doptr=1
[HOMING] dir=0 Zexpect=8132 Zactual=3778 Zdelta=4354 sens=3F
```

**Важно**: баг есть ТОЛЬКО в перекомпилированных версиях (из `Flint 2.77 src` и `Flint 2.77 (32bit) src`). Оригинальный бинарник из `!GENESIS\Flint 2.77` работает без ошибок.

Z_DIAG — отдельная программа для диагностики ISA-карты и оси Z **без запуска FLINT**. Запускается на целевой машине (Pentium III, DOS), выполняет серию тестов, результаты логирует в `Z_DIAG.LOG`.

---

## Целевая система

- **Компьютер**: Pentium III, 128 МБ RAM
- **ОС**: MS-DOS (загрузка из Windows 98 SE)
- **ISA-карта**: базовый адрес 0x300
  - 3× КР580ВИ53 = Intel 8253 (НЕ 8254!) таймера (PT0, PT1, PT2)
  - 2× 8255 PPI (PPI, PPI2)
  - RG (регистр управления моторами), LRG (лазер)
  - CMOS-счётчик точек
- **Компилятор**: Borland C++ 4.52 (16-bit, large model)

## Компиляция

### На DOS / Windows 9x (нативно)
```
COMPILE.bat
```
Ищет компилятор в `..\BC45\BIN\` или `C:\BC45\BIN\`. Результат: `ZDIAG\Z_DIAG.EXE`.

### На Windows NT+ (через DOSBox-X)
```
COMPILE.bat
```
Автоматически запускает DOSBox-X (ищет в `..\DOSBox-X\` или в PATH). Результат: `ZDIAG_DDMMYY_HHMMSS\Z_DIAG.EXE` + `RUN.bat`.

### Запуск на целевой машине
Скопировать каталог `ZDIAG_*` на машину, запустить `RUN.bat` (или `Z_DIAG.EXE` напрямую из DOS).

## Структура программы

### Принципы v2
- **4 фазы** в логическом порядке: окружение → идентификация железа → поведение железа → охота на баг
- `hw_init()` перед каждым тестом (гарантирует чистое состояние)
- **Нет Read-Back** в основном потоке (только тест 5, с немедленным `hw_init()` после)
- Вся проверка счётчиков — через **замер тайминга** (не write+readback, который бесполезен на 8253)
- Тесты пронумерованы 1-18 без пропусков

### Утилитарные функции

| Функция | Описание |
|---------|----------|
| `LOG(fmt, ...)` | Вывод на экран + в Z_DIAG.LOG |
| `read_counter(base, ch)` | Чтение 8253 счётчика (latch + lo/hi) |
| `read_sensors()` | Чтение PPI PB (датчики + PB6/PB7) |
| `esc_pressed()` | Проверка ESC (аварийная остановка) |
| `hw_detect()` | Обнаружение ISA-карты (K1=150 readback) |
| `hw_init()` | Полная инициализация (все mode CW, RG, PPI) |
| `hw_stop()` | Аварийная остановка моторов + OSC |
| `wait_z_done()` | Ожидание PB6 (Z done) с проверкой датчиков |
| `home_z(*steps)` | Хоминг только Z (2-фазный с дебаунсом). Всегда шлёт mode CW. |
| `home_all(reinit)` | Хоминг всех 3 осей (копия move_sen(0)). reinit=0 — точная реплика FLINT. |
| `move_xyz(x, y, z)` | Одновременное XYZ движение (копия move()) |
| `move_z(steps, k2z)` | Движение Z на заданное расстояние |
| `measure_k2z_timing(val)` | Замер реального периода K2z через тайминг (без mode CW — тест flip-flop) |
| `run_points(...)` | 180-точечный цикл обработки (9 слоёв × 20 точек) |
| `instrumented_homing(reinit, axes, *z_steps, *z_loop)` | Хоминг с записью точки срабатывания Z-датчика. Параметр axes — маска осей. |

## Параметры тайминга (по умолчанию)

Взяты из реального лога `FLINT32-12.03.2026_1.LOG`, строка `[PROC] CFG`:

| Параметр | Значение | Откуда |
|----------|----------|--------|
| `l_period` | 35 мс | Период лазера |
| `dT` | 6 мс | Коррекция CPU timing |
| `step_freq` | 1000 Гц | Макс. частота шагов |
| `f_const` | 0 | Переменная скорость (не const) |

Эти значения используются в `calc_k2()` и `seg_time()` для тестов 13 и 14 (XYZ-движение). Можно изменить через меню `S` перед запуском тестов.

## Безопасность

- Лазер **ВЫКЛЮЧЕН** (LRG=0, PC5=0, PC6=1)
- Все моторные тесты начинаются с хоминга к датчикам
- **ESC** — аварийная остановка в любом тесте
- Расстояния в пределах нормальных рабочих: Z≤8580 шагов (макс. Zscale=16384)
- Концевые датчики проверяются во время каждого движения
- Без ISA-карты программа работает (WARNING в логе, не блокирует)

## Интерпретация результатов

### Тест 5 показывает 8253 (Read-Back corrupt):
→ Подтверждено: таймеры 8253, не 8254. Write+readback бесполезен. Все проверки — через тайминг.

### Тест 9 показывает PB6 одноимпульсный:
→ Нужно перепроектировать wait_z_done() для работы с коротким импульсом.

### Тест 11 показывает spread > 20:
→ **Датчик Z нестабилен.** Все моторные тесты SUSPECT.

### Тест 14 воспроизводит step loss в Run 2:
→ **Баг подтверждён.** Смотреть лог Run 2: counter state, Z-steps vs expected.

### Тест 15 показывает spread Z+XY >> Zonly:
→ **XY-моторы мешают Z.** Вибрация или шум на датчиках.

### Тест 16 показывает noise events:
→ **Z-датчик шумит при XY-движении.** Нужен дебаунс в фазе C move_sen().

### Тест 17 показывает ratio > 10:
→ **K2z FLIP-FLOP CORRUPTED.** Запись K2z=100 → фактически K2z=25600. Фикс: re-send mode CW перед каждым move_sen().

### Тест 17 показывает ratio ≈ 1:
→ Flip-flop OK, причина в другом. Смотреть тесты 15, 16, 18.

### Тест 18 показывает drift > 10:
→ **Гравитационный дрейф.** Не отключать питание моторов между прогонами.

### Все тесты OK:
→ Баг рандомный/редкий. Повторить тест 14 многократно. Проверить различия compile-опций оригинала и recompile.
