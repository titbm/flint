# Информация об аппаратной части FLINT

## Источники

Схемы получены со сканов оригинальных документов:
- **FLINT+ Counter/Interface Card (f.cc.01)** — ISA-карта, July 22, 2000
- **FLINT Drawing Check Unit (fl.d.01)** — внешний блок, July 22, 2000
- BOM f.cc.01 — March 15, 1999
- BOM fl.d.01 — November 1, 1999

**Примечание:** Схемы описывают базовую версию платы (1× таймер, 1× PPI).
Текущая плата расширена до 3× таймеров (PT0, PT1, PT2) и 2× PPI (PPI, PPI2).
Дополнительные чипы подключены аналогично, через свободные выходы дешифраторов D9, D10.

---

## 1. Схема ISA-платы

**Статус: ПОЛУЧЕНА ✓**

Схема f.cc.01 прочитана. Ключевые цепи (для базовой версии, 1 таймер D19):

- **D19 (КР580ВИ53 = 8253)** — таймер, 3 канала
- OUT0 → R7 (240 Ом) → VT2 (КТ668А) → **OUT1** на разъём → внешний блок → шаговый мотор
- OUT2 → R11 (240 Ом) → VT3 (КТ668А) → **OUT2** на разъём → внешний блок → шаговый мотор
- GATE0, GATE1, GATE2 — управляются через логику (PMEN, SHEN) с подтяжкой 100 кОм
- CLK — от программируемого делителя (4× CD4017 + 4× CD4066), тактируемого BCLK ISA

- **D20 (КР580ВВ55А = 8255)** — PPI, подключён к разъёму X3
- **RG (D21C + логика)** — управляющий регистр через D-триггер К561ТМ2

Для расширенной платы: PT1 (0x304), PT2 (0x314), PPI2 (0x310) подключены к тем же шинам данных через буфер D3 (74ALS245), выбираются дешифраторами D9/D10.

---

## 2. Каскадирование счётчиков 8254

**Статус: ЧАСТИЧНО ОТВЕЧЕН ✓**

Софт загружает 6 значений для 3 осей в два чипа 8254:

```
PT0 (0x300):  ch0=K2x   ch1=dX    ch2=K2z
PT1 (0x304):  ch0=K2y   ch1=dY    ch2=dZ
```

### Ответы из схемы (базовая версия, 1 таймер):

- **CLK каждого K2-счётчика**: от программируемого делителя частоты на ISA-карте (4× CD4017 + 4× CD4066 + CD4013). Частота задаётся значением K1 из PPI PA.
- **GATE-входы**: управляются через логику (не +5V постоянно). Сигнал PMEN (вероятно Power/Motor ENable) от RG через D-триггер D21 и логические элементы подаётся на GATE. Подтяжка 100 кОм. **Это подтверждено Z_DIAG логом**: K2z не считает без ZON.
- **Режимы**: устанавливаются программно (control word через порт +3). Из кода: K2 в mode 2 (rate generator, 0xB4), d-counters в mode 4 (software triggered, 0x78/0xB8).
- **Каскадирование**: OUT ch0 (K2x) → выход на внешнюю плату (OUT1 через VT2 КТ668А) и **предположительно** CLK ch1 (dX). OUT ch2 (K2z) → выход на внешнюю плату (OUT2 через VT3 КТ668А) и **предположительно** CLK dZ на PT1.

### Ещё не подтверждено:
- Точная связь OUT K2z → CLK dZ (для расширенной платы, между PT0 и PT1)
- Подключение GATE для добавленных чипов PT1, PT2

## 3. Роль битов Motor Enable (RG D3-D5)

**Статус: ОТВЕЧЕН ✓**

Биты XON/YON/ZON (D3/D4/D5) регистра RG (0x30C) — **вариант (a): на вход GATE счётчика 8254**.

Из схемы: RG → D21 (К561ТМ2, D-триггер) → логика → GATE входы таймера D19, с подтяжкой 100 кОм.

Подтверждено Z_DIAG логом (TEST 1):
- K2z НЕ считает без ZON=1 → GATE закрыт
- K2z НЕ считает без OSC → нет CLK
- K2x считает БЕЗ XON → у K2x (ch0) GATE подключён иначе (возможно, постоянно HIGH или от другого бита RG)

**Вывод для кода:** Счётчик начинает считать ТОЛЬКО после записи в RG с установленным битом мотора. Это значит, что шаги НЕ теряются при загрузке — таймер ждёт включения мотора. Порядок в FLINT (загрузить таймеры → включить моторы) — корректен.

## 4. Сигналы PB6 и PB7 (PPI Port B, биты 6-7)

**Статус: ЧАСТИЧНО ОТВЕЧЕН**

В коде написано:
```
PB6: "IRQ2" (1 - event)    ← используется как флаг завершения Z
PB7: "IRQ3" (1 - event)    ← используется как флаг завершения XY
```

### Что видно из схемы:
На PPI (D20) PB6 и PB7 подключены ко входам с ISA-карты. По маркировке на схеме: сигналы промаркированы как внешние. Вероятнее всего, это выходы OUT счётчиков расстояния (dX, dZ), которые переходят из 0→1 при terminal count (mode 4: OUT goes HIGH after count reaches 0).

### Ещё не подтверждено:
- Какой именно OUT → PB6 и какой → PB7 (для расширенной платы)
- Механизм сброса: в коде используется `outportb(ADRPPI+2, PC1)` для сброса датчиков — возможно, PC1 сбрасывает и PB6/PB7 через внешнюю логику

## 5. Осциллятор

**Статус: ОТВЕЧЕН ✓**

### Из схем:

**Осциллятор — это ДВЕ ступени:**

**Ступень 1 — ISA-карта (f.cc.01):** Программируемый делитель частоты.
- K1 записывается в PPI PA (D20, 0x308) — это 8-бит число (7–255)
- PA выходы идут на 4× К561ИЕ10 (CD4017, декадные счётчики) + 4× К561КТ3 (CD4066, аналоговые ключи)
- CD4017 считает от BCLK (8 МГц ISA bus clock) и формирует тактовую частоту
- CD4066 — мультиплексор, выбирающий нужный делитель на основе K1
- Результат: тактовая частота F_osc, пропорциональная K1
- Бит D7 RG (OSCON/OSCOFF): управляет включением/выключением выхода делителя (подтверждено Z_DIAG TEST 1)

**Ступень 2 — внешний блок (fl.d.01):** Силовой усилитель.
- Сигнал OUT1/OUT2 с ISA-карты приходит через DB25
- D1 (К561ТМ2, CD4013, D-триггер) — формирует меандр (делитель на 2?)
- Транзисторные каскады: КТ342ВМ → КТ3107БМ → КТ645Б → мотор
- Трансформатор T1 в цепи обратной связи / развязки

### Формула частоты:
Точная формула пока неизвестна, но из Z_DIAG лога при K1=150 осциллятор активен.
Ожидаемо: F_step = F_osc / K2, где F_osc ~ K1 × (множитель).
**Необходим чистый TEST 6** (без порчи от Read-Back) для точного измерения.

## 6. Концевые датчики Z

**Статус: ЧАСТИЧНО ОТВЕЧЕН**

### Из схемы:
- PPI PB0–PB5 подключены ко входам через разъём X3 на ISA-карте
- **Нет фильтрации** (RC-фильтр, триггер Шмитта) на ISA-плате — входы PPI PB идут напрямую с разъёма
- Управление сбросом: PC1 (endSENSORS control) — вероятно, сбрасывает защёлку

### Подтверждено Z_DIAG логом:
- PB = 0x3F в покое → все 6 датчиков = 1 (свободны)
- **Логика: 0 = сработал, 1 = свободен** (подтверждено кодом: `stspb & 0x3f != 0x3f` → есть срабатывание)

### Ещё не подтверждено:
- Какие именно биты PB4/PB5 отвечают за Z (предполагается из маски `0x30`)
- Тип датчиков (оптический/механический)
- Расположение (верх/низ)

---

## 7. Шаговые моторы и драйверы

**Статус: ЧАСТИЧНО ОТВЕЧЕН ✓**

### Из схемы внешнего блока (fl.d.01):

**Драйверы — дискретные, на транзисторах:**
- Входной каскад: КТ342ВМ (маломощный NPN)
- Предрайвер: КТ3107БМ (PNP, средней мощности)
- Силовой ключ: КТ645Б (NPN, мощный, до 1А)
- Мощный каскад (1 канал): КТ829Г (составной Дарлингтон, до 3А)
- НЕ L297/L298, НЕ специализированная ИС — **полностью дискретная схема**
- Управление: L/R (резистивное), НЕ chopper/ШИМ
- Защитные диоды КД209Б на каждом электромагните

**D1 (К561ТМ2, CD4013)** — D-триггер, формирует полушаг или полный шаг из OUT-импульсов таймера.

### Ещё не подтверждено:
- Тип моторов (униполярный/биполярный), маркировка
- Шаг (1.8° или другой), ток обмотки
- Механика Z-оси (винт/ремень, шаг, масса)

## 8. Питание

**Статус: ЧАСТИЧНО ОТВЕЧЕН ✓**

### Из схем:
- **Логика ISA-карты**: +5V от ISA-шины (стандартно)
- **CMOS-счётчик**: батарея CR2032 (3V) через развязывающие диоды КД522Б — автоматическое переключение: питание от ISA когда включён, от батарейки когда выключен
- **Внешний блок**: питание +5V и -12V через DB25 с ISA. Силовой каскад: конденсаторы C41,C42 (100мкФ/30V), C43 (150мкФ/20V) — фильтрация. Мощные резисторы R113(1R0/0.5W), R115(51R/0.5W) — ограничение тока

### Ещё не подтверждено:
- Напряжение питания моторов (отдельный БП или с ISA 12V?)
- Просадки под нагрузкой

---

## Формат ответов

Для каждого пункта достаточно краткого ответа. Например:

> **2. Каскадирование**: OUT K2z → CLK dZ. GATE всех счётчиков привязан к +5V. Режим: K2 в mode 2, dXYZ в mode 0.

> **6. Датчики Z**: PB4/PB5, оптические фотопрерыватели, нормально-открытые, без фильтрации на плате.

Если есть **актуальная** схема расширенной платы (с 3 таймерами и 2 PPI) — она закроет оставшиеся вопросы. Если нет — Z_DIAG с исправленным кодом ответит на вопросы экспериментально.
