# CHANGELOG — Изменения после реставрации v2.77

Этот файл описывает **изменения и дополнения**, внесённые в ходе реставрации исходного кода по сравнению с оригинальной версией FLINT 2.77.

Оригинальные исходники v2.77 были утрачены. Исходный код в папке `Flint 2.77 src/` — результат обратного инжиниринга из бинарного EXE + исходников v2.61 + журнала изменений. Перечисленные ниже функции **отсутствуют в оригинальном EXE** и добавлены при реставрации.

---

## [v2.78] — Добавлено после реставрации

### 1. Увеличение памяти в 32-битных версиях

**Затронутые сборки:** `Flint 2.77 (WTCM32) src/`, `Flint 2.77 (BCC32) src/`

16-битная версия выделяет память через `farmalloc(farcoreleft() - 30000)`. В типичном DOS-окружении это ~500 КБ, что соответствует максимум ~80 000 точек.

32-битные порты используют каскадную аллокацию через `malloc()`:

| Попытка | Точек | Память |
|---------|-------|--------|
| 1-я | **8 000 000** | 48 МБ |
| 2-я (fallback) | **2 000 000** | 12 МБ |
| 3-я (fallback) | **400 000** | 2.4 МБ |

После аллокации выполняется верификация буфера (запись/чтение тестовых значений в начало, середину и конец).

---

### 2. Диагностическое логирование (TRACE / FLINT.LOG)

**Файлы:** `PIC.CPP`, `PROJ5.CPP`

Добавлена система диагностического логирования для удалённой отладки. При сборке с флагом `TRACE` (включён по умолчанию; отключается через `-DNOTRACE`) программа ведёт файл `FLINT.LOG` рядом с EXE.

В лог записываются:
- Запуск и завершение программы (`[INI]`, `[EXIT]`)
- Каждый шаг конфигурации (`[CONFIG]`)
- Каждый лазерный импульс (`[LPULS]`, `[SHOT]`, `[METAL]`) — номер цикла, состояние PT2, маска затворов, флаг `d_check`
- Результат каждого выстрела — показания датчиков `dsens`, список позиций с ошибкой `wrkpos`
- Перемещения столика: параметры `move()`, привязка к датчикам `[HOMING]`
- Запуск и завершение гравировки (`[START]`, `[STOP]`, `[PROC]`)
- Аппаратные записи в порты (`[HW]`)

Каждый вызов `tlog_write()` немедленно сбрасывается на диск (`fflush`), поэтому лог сохраняется даже при аварийном завершении.

### 3. Проверка включения лазерной помпы перед запуском гравировки

**Файл:** `PROJ5.CPP`, функция `dlgstart()`

В оригинальном EXE v2.77 запуск гравировки (`F5 → Ok`) не проверял, включена ли лампа накачки. Если оператор забывал включить насос в меню F1, программа начинала выполнять точки вхолостую — столик двигался, затворы открывались, но лазер не стрелял.

Добавлена проверка в начале обработчика кнопки «Ok»:

```c
if(!f_metal && !LCtl) { msgbox1("Laser is not enabled."); goto b; }
```

Если `LCtl == 0` (ни лампа накачки D0, ни затвор D1 не включены) и не активен metal-режим — отображается диалог «Laser is not enabled.» и запуск блокируется. Для metal-режима проверка не применяется, так как он не использует помпу.

### 4. Кнопка Shot — одиночный тестовый импульс

**Файл:** `PROJ5.H` (кнопка 28 в `btc_cfg`/`bnam_cfg`), `PROJ5.CPP`, `dlgcfg()` case 28

Добавлена кнопка **Shot** в диалог F1 (Configuration). Выполняет один одиночный лазерный импульс без запуска полного цикла гравировки. Предназначена для тестирования позиции фокуса, настройки gate delay и проверки оптического тракта.

Перед выстрелом определяется значение gate: если `f_gate` включён — используется текущий `gate_delay`, иначе gate=1 (минимальный, 10 мкс). Также запоминается, была ли помпа уже включена (`_pump_was_on = (LCtl != 0)`).

Последовательность:
1. Открывает защитные шторки (`PR_OPEN`) и электромагнитные затворы (`sh_open`)
2. Программирует PT2 безопасное значение gate=1
3. Включает насос `LCtl = PUMP_ON | SHUT_ON` (только если помпа не была уже включена)
4. Ждёт синхронизации с таймером PT2 (`while((read_pt2()-2) & 0xFFE0)`, с таймаутом)
5. Программирует PT2 рабочее значение gate, delay(1), сбрасывает gate обратно в 1
6. Выключает насос `LCtl = 0` (только если сам включал — чтобы не прерывать текущий режим обогрева)
7. Закрывает шторки

Насос включается до цикла синхронизации, потому что тактовый сигнал PT2 аппаратно зависит от бита PUMP_ON в регистре LRG.

В demo-режиме кнопка издаёт звуковой сигнал и ничего не делает.

### 5. Переименование кнопки `#` → `Reset`

**Файл:** `PROJ5.H`, массив `bnam_cfg`

В оригинальной программе кнопка сброса счётчика вспышек и даты обслуживания (раздел «Statistics» в F1) называлась `#`. Переименована в **Reset** для ясности.

### 6. Metal mode — режим гравировки металла

**Файлы:** `PROJ5.H`, `PROJ5.CPP`, `PIC.CPP`

Добавлен переключатель **metal mode** (кнопка-чекбокс в диалоге F1, case 29). При включении:
- Выключает помпу (`LCtl = 0`)
- Сбрасывает `f_gate = 0`
- Программирует PT2 с минимальным gate=1

При выключении — просто сбрасывается флаг `f_metal`, без аппаратных действий.

**Отличие от обычного режима:**

| Аспект | Обычный режим (стекло) | Metal mode |
|--------|------------------------|------------|
| Помпа в покое | Включена постоянно | Выключена (`LCtl = 0`) |
| Помпа при выстреле | Уже включена до `lpuls()` | Включается внутри `lpuls()` перед каждым импульсом, выключается сразу после |
| Выстрел | Через PT2 (gate delay) | Через PT2 (gate delay), но помпа управляется per-point |

В `lpuls()` при `f_metal`: перед каждой попыткой синхронизации с PT2 включается помпа (`LCtl = PUMP_ON | SHUT_ON`), после импульса сразу выключается (`LCtl = 0`). Это необходимо, потому что тактовый сигнал PT2 аппаратно зависит от бита PUMP_ON.

В `process()` обе ветки (metal и обычная) вызывают одну и ту же `lpuls(flag)` — логика retry/EM-шаттеров одинаковая.

Проверка насоса перед запуском (п. 3) для metal mode отключена — он не требует помпы в состоянии ожидания.
