# Справочник по лог-файлу FLINT.LOG

Лог пишется в файл `FLINT.LOG` рядом с exe. Каждая строка начинается с тега `[СЕКЦИЯ]`.

---

## Структура сеанса (порядок записей)

```
=== FLINT 2.77 Trace Log Start ===
[HW] LRG <- 00 (init)           ← инициализация оборудования
[INI] ...                        ← параметры оборудования
[UI] MAIN btn=...                ← действия пользователя (кнопки меню)
[DATA] LOAD ...                  ← загрузка файла данных
[CONFIG] ...                     ← настройки конфигурации
[DLGSTART] ...                   ← диалог запуска
[START] ...                      ← старт обработки
[PROC] ...                       ← начало process()
[HOMING] ...                     ← калибровка осей (move_sen)
  [PT] ... [MOV] ... [MOV] DONE  ← цикл обработки точек (повторяется N раз)
  [LPULS] ...                    ← лазерный импульс
[STOP] ...                       ← остановка (ESC или TABLE_END)
[GLLOAD] ... [HOMING] ...        ← загрузка/выгрузка стекла (F4)
[HW] LRG <- 00 (shutdown)       ← завершение
=== Trace Log End ===
```

---

## Записи по секциям

### `[HW]` — Аппаратные операции

| Запись | Значение |
|--------|----------|
| `[HW] LRG <- XX (init)` | Запись в регистр лазера. `00` = всё выключено |
| `[HW] LRG <- XX (shutdown)` | Завершение программы, всё выключено |
| `[HW] PT2 ch0=XXXX (lo=XX hi=XX) ch1=X (XXus)` | Настройка таймера лазера PT2: ch0=период импульса, ch1=задержка вентиля |
| `[HW] SH_OPEN pos=XXXX PB=XX shd=XX` | Открытие электромагнитных затворов (EM). `pos` — битовая маска каналов, `shd` — задержка (мс) |
| `[HW] SH_CLOSE all-off` | Закрытие всех EM-затворов |
| `[HW] PPI2 PR_OPEN` | Снятие защиты (разрешение движения моторов) |
| `[HW] PPI2 PR_CLOSE` | Включение защиты (блокировка моторов) |
| `[HW] RG OUT XX fxy=N fz=N` | Запись в регистр управления моторами (ADRRG). `XX` — байт управления, `fxy` — движутся ли X/Y, `fz` — движется ли Z |
| `[HW] RG STOP XX sens=XX` | Аварийная остановка моторов (концевик сработал во время хода) |
| `[HW] RG RECOVER XX` | Откат от концевика |
| `[HW] RG END XX` | Завершение отката от концевика |

### `[INI]` — Инициализация

```
[INI] hw=1 K1=150 lper=35 gate=19 shd=100 dchk=1
```
| Поле | Значение |
|------|----------|
| `hw` | 1 = оборудование найдено, 0 = демо-режим |
| `K1` | Делитель частоты (PPI порт A, ожидается 150) |
| `lper` | Период лазера (l_period) |
| `gate` | Задержка вентиля (gate_delay) |
| `shd` | Задержка затвора (sh_delay, мс) |
| `dchk` | 1 = проверка датчиков разрушения включена |

### `[UI]` — Действия пользователя

```
[UI] MAIN btn=5 (F5 Start) pkey=0
```
| Поле | Значение |
|------|----------|
| `btn` | Номер кнопки (0=F9 XYZ, 1=F1 Config, 2=F2 Convert, 3=F3 DatLoad, 4=F4 GlsLoad, 5=F5 Start, 6=F10 Exit) |
| `pkey` | Код клавиши (0 = нажата мышью) |

```
[UI] GLASSLOAD UP     — F4 → PgUp (поднять стекло к датчику)
[UI] GLASSLOAD DOWN   — F4 → PgDn (опустить стекло)
```

### `[DATA]` — Загрузка данных

```
[DATA] LOAD file=cube.bft points=1000
```
Загружен файл `cube.bft` с 1000 точками.

### `[CONFIG]` — Конфигурация

```
[CONFIG] PUMP ON                          — помпа включена
[CONFIG] PUMP OFF                         — помпа выключена
[CONFIG] OK lper=35 gate=19 sfq=1000 shd=100 const=0 metal=0
                                          — применены параметры
[CONFIG] EXIT LCtl=3 gate_val=19 (gate reset to 1)
                                          — выход из конфигурации
```
| Поле | Значение |
|------|----------|
| `lper` | Период лазера |
| `gate` | Задержка вентиля |
| `sfq` | Частота шагов (step_freq) |
| `shd` | Задержка затвора |
| `const` | 0 = переменная скорость, 1 = постоянная |
| `metal` | 0 = стекло, 1 = металл |
| `LCtl` | Состояние лазерного регистра (бит 0 = помпа, бит 1 = заслонка) |

### `[DLGSTART]` — Диалог запуска (нажат F5)

```
[DLGSTART] ENTER lastn=0 xcur=0 ycur=0 zcur=0 doptr=0 fptr=0 demo=1
```
| Поле | Значение |
|------|----------|
| `lastn` | Последняя обработанная точка (0 = с начала) |
| `xcur/ycur/zcur` | Текущие координаты осей |
| `doptr` | 1 = оси уже откалиброваны (хоминг выполнен) |
| `fptr` | 1 = продолжение с точки (continuation), хоминг нужен |
| `demo` | 1 = демо-режим (без лазера и моторов) |

### `[START]` — Запуск обработки

```
[START] st=0 fstart=2 metal=0 dchk=1 LCtl=3 gate=19
[START] PRE-PROC xcur=0 ycur=0 zcur=0 doptr=0 fptr=0
```
| Поле | Значение |
|------|----------|
| `st` | Стартовая точка (start_nmb) |
| `fstart` | Режим старта (0=с начала, 1=с точки, 2=продолжение) |

```
[START] POST-PROC k=1 lastn=0 xcur=22450 ycur=690 zcur=9096 doptr=1
```
Результат после завершения process():
- `k=1` — нормальное завершение, `k=0` — прервано (ESC или ошибка)
- `lastn` — последняя точка (0 = прошли все)

### `[PROC]` — Обработка (process)

```
[PROC] START n=0/1000 metal=0 dchk=1 lper=35 gate=19 shd=100
```
Начало обработки: `n=стартовая/всего` точек.

```
[PROC] CFG xptr=640 yptr=280 zptr=150 zsize=400 nglas=1.5060 step=10 sf=1000 dT=6 fc=0 demo=1
```
| Поле | Значение |
|------|----------|
| `xptr/yptr/zptr` | Смещение точки фокуса (из конфига) |
| `zsize` | Размер по Z (глубина стекла × 10) |
| `nglas` | Показатель преломления стекла |
| `step` | Базовый шаг (единиц координат на 1 raw-шаг) |
| `sf` | Частота шагов (Гц) |
| `dT` | Параметр рассинхронизации |
| `fc` | 0=переменная скорость, 1=постоянная |
| `demo` | 1 = без лазера/моторов |

```
[PROC] CALC refr_ofs=1343 z_base=9507 (11000-zptr-refr_ofs)
```
Расчёт преломления: `refr_ofs` = коррекция на преломление, `z_base` = база координат Z.

```
[PROC] CHK doptr=0 fptr=0 xcur=0 ycur=0 zcur=0 => HOMING
[PROC] CHK doptr=1 fptr=0 xcur=22450 ycur=690 zcur=9096 => SKIP
```
Проверка: делать хоминг или пропустить.
- **HOMING** — если `doptr=0` (оси не калиброваны) или `fptr=1` (продолжение)
- **SKIP** — если `doptr=1` и `fptr=0` (оси уже откалиброваны, не continuation)

```
[PROC] RESUME xcur=23300 ycur=200 zcur=200 doptr=1 fptr=0 st=0
```
Координаты после хоминга/перед началом цикла точек.

### `[HOMING]` — Калибровка осей (move_sen)

```
[HOMING] dir=0 Zexpect=9384 Zactual=16384 Zdelta=-7000 sens=FF
```
| Поле | Значение |
|------|----------|
| `dir` | 0 = вверх (к датчику, стекло вверх), 1 = вниз |
| `Zexpect` | Ожидаемое расстояние до Z-датчика = значение zcur перед хомингом |
| `Zactual` | Реальное кол-во шагов Z до срабатывания датчика (= 16384 − остаток счётчика dZ) |
| `Zdelta` | Zexpect − Zactual. **0 ≈ норма**. Большое положительное = датчик сработал рано (ложное срабатывание). 16384 в Zactual = demo-режим (моторы не двигались) |
| `sens` | Байт PPI+1 (состояние всех датчиков): биты 0-1=X, 2-3=Y, 4-5=Z концевики, 6=Z done, 7=XY done. FF = все высокие (демо или все сработали) |

**После хоминга dir=0**: `xcur=23300, ycur=200, zcur=200` (домашняя позиция).

### `[GLLOAD]` — Загрузка/выгрузка стекла

```
[GLLOAD] dir=0 (UP)     — стекло поднимается к датчику
[GLLOAD] dir=1 (DOWN)   — стекло опускается
```
За этой записью всегда следует `[HOMING]` с результатом move_sen.

### `[PT]` — Точка данных

```
[PT] i=0 raw=86,86,86 xyz=22774,366,9420 d=-526,166,9220
```
| Поле | Значение |
|------|----------|
| `i` | Номер точки (0-based) |
| `raw` | Сырые координаты из файла (iTabl[3i], iTabl[3i+1], iTabl[3i+2]) |
| `xyz` | Целевые координаты после преобразования (куда двигаться) |
| `d` | Дельта от текущей позиции: `dx=xyz.x-xcur`, `dy=xyz.y-ycur`, `dz=xyz.z-zcur` |

**Формулы преобразования:**
- `x = 23500 - xptr - raw_x`
- `y = yptr + raw_y`
- `z = 11000 - zptr - refr_ofs - raw_z`

### `[MOV]` — Команда перемещения

```
[MOV] dx=-526 dy=166 dz=9220 tseg=264 k2x=1755 k2y=5562 k2z=100
```
| Поле | Значение |
|------|----------|
| `dx/dy/dz` | Дельта перемещения (шаги). Знак: + = вверх/вправо, − = вниз/влево |
| `tseg` | Время сегмента (t_seg, единицы 10µс). Определяется самой длинной осью |
| `k2x/k2y/k2z` | Делители частоты для каждой оси (записываются в 8254). 0 = ось не движется |

### `[VFY]` — Верификация счётчиков (только в новых сборках)

```
[VFY] K2x=1755 dX=526 K2y=5562 dY=166 K2z=100 dZ=9220
```
Перечитывание 8254 **до старта моторов**. Показывает, что реально записалось в счётчики:
| Поле | Значение |
|------|----------|
| `K2x/K2y/K2z` | Делитель частоты (чем больше — тем медленнее ось) |
| `dX/dY/dZ` | Количество шагов (должно совпадать с `dx/dy/dz` из `[MOV]`) |

**Если dZ ≠ abs(dz из [MOV])** — значит в счётчик записалось не то, что ожидалось. Признак сбоя шины или помехи.

### `[MOV] DONE` — Перемещение завершено

```
[MOV] DONE sens=FF Xd=65535 Yd=65535 Zd=65535 dz_prog=9220 cur=23300,200,200->22774,366,9420
```
| Поле | Значение |
|------|----------|
| `sens` | Байт PPI+1 после остановки (FF = все биты 1 = норма или demo) |
| `Xd/Yd/Zd` | Остатки счётчиков после хода (65535 = demo, не работают). На реальном железе: 0 = досчитал полностью |
| `dz_prog` | **Ключевое**: сколько шагов Z было запрограммировано для этого хода. Знак показывает направление |
| `cur=X1,Y1,Z1->X2,Y2,Z2` | Координаты ДО и ПОСЛЕ этого хода |

**Важно**: `dz_prog` — это `dz` из `[MOV]`, т.е. запрограммированное перемещение Z. На реальном железе `Zd` показывает сколько шагов **осталось** в счётчике — обычно 0.

### `[LPULS]` — Лазерный импульс

```
[LPULS] CALL flag=1 dchk=1 metal=0       — вызов: flag=1 = первая точка (открыть затвор)
[LPULS] demo ok fl=40471                  — демо: импульс засчитан, fl = счётчик вспышек
[LPULS] ok i=0 dsens=0200 mask=0200 fl=N — реальный: i=попытка, dsens=датчик разрушения
[LPULS] retry i=2 dsens=0000 wrk=0200    — повтор: датчик не подтвердил попадание
[LPULS] FAIL i=10 wrkpos=0200 dsens=0000 — ошибка: 10 попыток — датчик не сработал
[LPULS] ESC i=3 wrkpos=0200              — прервано по ESC во время импульса
```

### `[METAL]` — Металлический режим (прямой импульс через PC6)

Аналогично `[LPULS]`, но без PT2 таймера:
```
[METAL] CALL flag=1 dchk=1
[METAL] demo ok fl=100
[METAL] ok i=0 dsens=0200 mask=0200 fl=N
```

### `[STOP]` — Остановка обработки

```
[STOP] TABLE_END zcur=9096 (Z до датчика: 9096 шагов)
```
Все точки обработаны. `zcur` — текущая Z-координата = расстояние, которое Z должен проехать вверх до датчика.

```
[STOP] ESC pt=155/1000 zcur=9384 (Z до датчика: 9384 шагов)
```
Прервано по ESC на точке 155 из 1000.

### `[MOVE] ERR` — Ошибка перемещения (концевик)

```
[MOVE] ERR sens=2F x=22774 y=366 z=9420 cur=23300,200,200
```
Во время хода сработал концевик (биты 0-5 PPI+1). Программа отъедет назад (RECOVER) и остановится.

### `[DIAG]` — Диагностика Z (F7)

```
[DIAG] Z-TEST START
[DIAG] Z-TEST i=0: move down dz=5000
[DIAG] Z-TEST i=0: homing...
[DIAG] Z-TEST i=0 RESULT Zexp=5000 Zact=XXXX Zdelta=YYYY sens=ZZ
```
Специальный тест повторяемости Z-оси (вызывается по F7). Двигает Z вниз, затем хоминг — повторяет несколько раз и сравнивает.

## Как читать лог для диагностики

### Проверка: сделал ли Z хоминг?

Ищите `[PROC] CHK`:
- `=> HOMING` — хоминг выполнен, далее будет `[HOMING]`
- `=> SKIP` — хоминг пропущен (doptr=1, fptr=0)

### Проверка: на какой Z-глубине рисовал?

Ищите строки `[MOV] DONE` с `dz_prog≠0` — это моменты, когда Z двигалась. `cur=...,Z1->...,Z2` показывает Z до и после.

### Проверка: нормально ли завершился хоминг?

В `[HOMING]`:
- **Zdelta ≈ 0** → Z доехал до датчика штатно
- **Zdelta >> 0** → датчик сработал рано (ложное срабатывание, Z не доехал)
- **Zactual = 16384** → либо demo-режим, либо датчик не сработал за весь бюджет 16384 шагов

### Проверка: совпала ли запись в счётчик?

В `[VFY]`: `dZ` должен быть равен `abs(dz)` из предшествующего `[MOV]`. Если не совпадает — сбой записи в 8254.

### Типичный цикл одной точки

```
[PT] i=5 raw=266,86,86 xyz=22594,366,9420 d=-36,0,0    ← куда ехать
[MOV] dx=-36 dy=0 dz=0 tseg=2 k2x=177 k2y=0 k2z=0     ← расчёт хода
[VFY] K2x=177 dX=36 K2y=0 dY=0 K2z=0 dZ=0              ← верификация
[HW] RG OUT 0E fxy=1 fz=0                                ← старт моторов
[MOV] DONE sens=FF Xd=0 Yd=0 Zd=0 dz_prog=0 ...        ← завершение
[LPULS] CALL flag=0 dchk=1 metal=0                       ← импульс
[LPULS] ok i=0 dsens=0200 mask=0200 fl=40476             ← успешно
```
